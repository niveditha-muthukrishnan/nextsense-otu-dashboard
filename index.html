Methodology & Definitions — How These Numbers Were Calculated

OTU Cohort
Users who fired change_onboarding_completed with isOnboardingCompleted=true since Feb 9, 2026. 456 users after excluding internal accounts (aligned with CEO report: user_cohort='Internal', email contains 'nextsense', plus @test.com/@example.com).
Sleep Night (for engagement categories)
A unique calendar date with a qualifying session_statistics event (type=sleep, duration >90 min). Aligned with CEO report methodology. One night = one date regardless of how many sessions that night.
Engagement Categories (mutually exclusive)
Never Active: 0 qualifying sessions ever.
Resurrected: Had a gap >15d between sessions, then returned in last 7d.
Highly Active: Avg 4+ sessions/week over rolling 4-week window (pro-rated for newer users).
Active: ≥1 session in last 7 days (but <4/week avg).
At-Risk: 8–15 days since last session.
Churned: >15 days since last session.
Qualifying Session (engagement + sleep metrics)
session_statistics event with type=sleep (not nap) and cumulativeSessionDuration >90 min. Aligned with CEO report. Used for both engagement categories and sleep metrics.
Deep Sleep
Source: slowWaveMinutes on session_statistics. Prefers slowWaveMinutes (CEO report), falls back to totalDeepSleepMinutes. Measured in minutes.
Boost (Audio Stimulation)
Source: audioStimulation on session_statistics. Prefers audioStimulation (0-12, CEO report), falls back to slowWaveCount (raw stim count).
Consistent Sleeper (4/7)
User had qualifying sessions on 4+ of the last 7 nights (pro-rated if activated <7 days ago).
Internal Exclusion (aligned with CEO report)
(1) user_cohort='Internal' (case-insensitive), (2) email contains 'nextsense' (substring match on both $email and email properties), (3) @test.com/@example.com domains (additional safety). Identified from Engage API profiles, then all events for those user IDs are removed.
